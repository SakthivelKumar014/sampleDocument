Introduction

RWS Application Programming Interface (API) by integrated web services called as Riya Web Services (RWS) which offers a means of accessing and using data from its hosted services. It provides a gateway access to all Airlines Tickets booked through Riya Travel as product for an external plug-in. This gateway may be used to develop a separate application, or be integrated with an existing tool of users.

1.Login Method

Parameters Value
TerminalID will be provided
Username will be provided
Password will be provided

1.1 Description

HTTP Basic Authentication is the simplest form of identification. This technique combines Terminal ID, Username and Password to form a single value and passes it through a special HTTP header known as Authorization where they are encoded with Base64. Login method will return the Token that needs to be echoed back in every subsequent request. In case of token expiry, the login method needs to be called again and the updated Token needs to be passed in every subsequent request.

Note: Once token is generated it valid till end of the day.


1.2 Authentication

HTTP Basic Authentication

Step 1 : Generate Base64 string with the following logic TerminalID*Username:Password

Step 2 : Apply the Base64 string in request header section as mentioned below

Key Value
Authorization Base64 string (e.g:UaJjLkFCT0AwMDcSsMDEqbHAScHChscAHWWFAMTI)

2.Availability

2.1 Description

Availability method provides available flight information along with the cheapest fare options. The sample request includes single AvailInfo query for Oneway journey. In case of Roundtrip, this field should be added with the return flight query. Once the requested query is parsed and validated, the booking engine start searching for the best option in all possible sources. If the search is successful it will respond the flight details in ItineraryFlightList object.

2.2 Data Format and Details

Field Name Data Type Value
AgentID String Your Agent ID
TerminalID String Your Terminal ID
Username String Your Username
AppType String Default Value API
Version String API version
TripType String Trip Type shows the type of booking. It may be an O-Oneway or R-Roundtrip.
AirlineID String Two letter Airline Code. In case of empty values system consider as all airlines.
DepartureStation String 3 Letter IATA Departure Airport code
ArrivalStation String 3 Letter IATA Arrival Airport code
FlightDate Date Travel date should be in the following format (yyyymmdd)
FarecabinOption String Cabin Identification E- Economy, P- Premium Economy, B- Business and F- First
FareType String Fare Identification N- Normal Fare, C- Corporate Fare and R- Retail Fare
OnlyDirectFlight Boolean True –System responds only direct flight. False – System responds all the flights.
AdultCount Integer Minimum of 1 and Maximum up to 9
ChildCount Integer Total no of Adults and child can be maximum 9
InfantCount Integer Minimum of 1 and Maximum up to 4. Infant alone not allowed to travel

Request

                  
  {
    "AgentInfo": {
      "AgentId": "xxxxxxxxxxx",
      "TerminalId": "xxxxxxxxxxx",
      "UserName": "xxxxxx",
      "AppType": "API",
      "Version": "V1.0"
    },
    "TripType": "O",
    "AirlineID": "",
    "AvailInfo": [
      {
        "DepartureStation": "BOM",

        "FlightDate": "20221114",
        "FarecabinOption": "E",

        "OnlyDirectFlight": false
      }
    ],
    "PassengersInfo": {
      "AdultCount": "1",
      "ChildCount": "0",
      "InfantCount": "0"
    }
  }

3.Fare Rules

2.1 Description

GetFarerule method is used to get the fare rules and condition of a specific flight option. It contains the fare basis code and the relevant details associated with it, such as restrictions and penalties.

2.2 Data Format and Details

Field Name Data Type Value
AgentID String Your Agent ID
TerminalID String Your Terminal ID
Username String Your Username
AppType String Default Value API
Version String API version
FlightID String Pass the same value from Availability response.
TrackId String Unique reference Id from Availability response.
